# 천둥마켓 프로젝트 안내

## 이 프로젝트는 어떤 곳인가요
- 개발 **연습용** 폴더입니다. 마음껏 시도하고 실험해도 괜찮습니다.
- 사용 기술: **Next.js**(웹사이트를 만드는 도구), **Supabase**(회원가입·데이터 저장 등을 처리하는 서버).
- 배포 방식: **GitHub에 올리면(push) 자동으로 Vercel이 인터넷에 배포**합니다.
  즉, 코드를 GitHub에 업로드하면 잠시 뒤 실제 사이트에 반영됩니다.

## 대화하고 일하는 방식 (꼭 지켜주세요)

### 1. 쉬운 말로 설명하기
- 사용자는 비개발자입니다. 개발 용어(빌드, 커밋, 마이그레이션 등)는
  **풀어서 쉬운 비유로** 설명합니다.
  예) "커밋" → "지금까지 한 작업을 저장하는 것", "배포" → "인터넷에 실제로 올리는 것".

### 2. 무엇을 하는지 자주 알려주기
- 지금 무슨 작업을 진행 중인지, 왜 그걸 하는지 **자주, 짧게** 알려줍니다.
- 작업이 길어지면 중간중간 "지금 ~하는 중이에요" 하고 상황을 공유합니다.

### 3. 시킨 것만 단계적으로
- 사용자가 **지시한 것만** 수행합니다. 시키지 않은 기능을 미리 만들지 않습니다.
- 한 번에 다 하지 말고 **한 단계씩** 진행하며, 다음 단계로 넘어가기 전에 확인합니다.

### 4. 할 수 있는 건 최대한 직접 처리
- 코드 작성, 파일 수정, 명령 실행 등 **할 수 있는 일은 최대한 직접** 끝냅니다.
- **사람이 직접 해야만 하는 일**(예: 웹사이트에서 버튼 클릭, 비밀번호·키 입력,
  계정 연결, 결제 등)만 따로 모아서 **번호를 매겨 쉽게 안내**합니다.
  - 어디를 눌러야 하는지, 무엇을 입력해야 하는지 구체적으로 알려줍니다.

### 5. 개발이 끝나면 바로 확인할 수 있게
- 작업을 마치면 **결과를 바로 눈으로 확인할 수 있도록 열어줍니다.**
  - 화면 변경이면 개발 서버를 실행하고 브라우저 주소(예: http://localhost:3000)를 안내합니다.
  - 배포 결과면 확인할 수 있는 링크를 알려줍니다.

## 개발 환경 참고
- 이 폴더의 Next.js는 일반적인 버전과 다를 수 있습니다.
  코드를 쓰기 전에 `node_modules/next/dist/docs/` 안의 관련 문서를 먼저 확인합니다. (위 AGENTS.md 참고)
- 명령 실행 시 `node`, `gh` 등이 PATH에 등록되어 있지 않을 수 있으니 주의합니다.
